# Fishbone (Ishikawa) Categories for Business Analysis

Adapted from manufacturing's 6M categories (Man, Machine, Method, Material, Measurement, Mother Nature) to business analysis context. Use these categories in Step 4 when building the Ishikawa diagram. Populate at least 4 of 6 categories before concluding your analysis.

---

## People
Causes related to human factors - skills, capacity, roles, and communication.

| Example Cause | Description |
|--------------|-------------|
| Unclear roles and responsibilities | No RACI defined; multiple people think someone else owns the task |
| Insufficient training | Staff lack skills to perform the process correctly |
| High turnover | Knowledge walks out the door with departing employees |
| Key person dependency | Process only works when one specific person is available |
| Communication gaps | Information doesn't flow between teams or shifts |
| Resistance to change | Staff revert to old ways because new process wasn't adopted |

## Process
Causes related to workflow design - steps, handoffs, bottlenecks, and documentation.

| Example Cause | Description |
|--------------|-------------|
| Missing process steps | Critical validation or check is absent from the workflow |
| Undocumented handoffs | Work passes between teams without a defined protocol |
| Bottleneck at approval | Single approver creates a queue that delays everything |
| No escalation path | When something goes wrong, there's no defined response |
| Redundant steps | Same check performed multiple times by different people |
| Inconsistent execution | Different teams follow different versions of the process |

## Technology
Causes related to systems - availability, performance, integration, and functionality.

| Example Cause | Description |
|--------------|-------------|
| System downtime | Unplanned outages disrupt the process |
| Integration failures | Systems don't communicate correctly; data gets lost in transit |
| Data sync lag | One system has stale data because sync is batch, not real-time |
| Missing functionality | System doesn't support a required operation, forcing workarounds |
| Poor UX causing workarounds | Interface is so difficult that users bypass it |
| Legacy system limitations | Old system can't be modified to support new requirements |

## Policy
Causes related to organizational rules - governance, compliance, approvals, and SLAs.

| Example Cause | Description |
|--------------|-------------|
| Outdated policies | Written policies conflict with actual practice |
| Regulatory non-compliance | Process doesn't meet current legal or regulatory requirements |
| Approval thresholds too rigid | Low-value items require the same approval as high-value ones |
| Missing SLAs | No agreed response times between internal teams |
| Conflicting policies across departments | Department A's rule contradicts Department B's rule |
| No exception handling rules | No defined path for non-standard situations |

## Environment
Causes related to organizational context - structure, resources, and external factors.

| Example Cause | Description |
|--------------|-------------|
| Organizational restructuring | Recent reorg left unclear ownership of processes |
| Budget constraints | Insufficient funding to maintain or improve the process |
| Competing priorities | Team is pulled between this process and other urgent work |
| Vendor dependency | External supplier controls a critical part of the process |
| Geographic distribution | Teams in different locations create coordination challenges |
| Timeline pressure | Deadlines force shortcuts that introduce errors |

## Data
Causes related to information - quality, availability, accuracy, and access.

| Example Cause | Description |
|--------------|-------------|
| Data quality issues | Incorrect, incomplete, or inconsistent data enters the process |
| Missing data fields | Required information is not captured at the source |
| Duplicate records | Same entity represented multiple times, causing confusion |
| No single source of truth | Multiple systems hold conflicting versions of the same data |
| Stale data | Decisions made on outdated information |
| Access permission gaps | People who need data can't access it; people who shouldn't can |
