# Newsletter Ideation

## How It Works

### Step 1: Clarify the Input

Ask which type of input the user is starting with:
- **Specific topic** ("AI agents in healthcare") - use SCAMPER + Intersection Discovery
- **Trend** ("AI replacing jobs") - use Time-Horizon Shifts + Contrarian Angle Generator
- **Vague direction** ("something about productivity") - use Jobs-to-be-Done first

### Step 2: Apply Frameworks

Generate 5-7 angles, each from a different framework. Label each with the framework used - this forces variety and prevents generating variations on the same theme.

**SCAMPER** - Modify the topic through 7 lenses:
- Substitute: replace a core element with something unexpected
- Combine: merge with an unrelated topic
- Adapt: apply a pattern from a different field
- Modify/Magnify: exaggerate one aspect to its extreme
- Put to other uses: show it applied in an unexpected context
- Eliminate: remove what everyone assumes is essential
- Reverse: argue the opposite of the consensus

**Jobs-to-be-Done** - Ask what the reader is hiring this content to do:
- Functional: help them accomplish a specific task
- Emotional: make them feel capable or validated
- Social: give them something worth sharing or discussing

**Contrarian Angle Generator** - Find the hidden assumption in the consensus view:
1. State the common belief
2. Identify the assumption everyone accepts but hasn't verified
3. Build the angle around that assumption being wrong or incomplete

**Time-Horizon Shifts** - Tell the same story at three time scales:
- This week (immediate action)
- This year (strategic implication)
- This decade (structural shift)

**Stakeholder Perspective Rotation** - Tell the same story from different viewpoints. Adapt these to your niche:
- Practitioners (builders, doers)
- Decision-makers (managers, leads)
- Strategists (executives, directors)
- End users (customers, adopters)

**Intersection Discovery** - Find where the topic intersects with an unrelated domain.
Example: "AI agents" x "supply chain logistics" = a piece most newsletters in either space won't write.

### Step 3: Format Each Angle

For each of the 5-7 angles output:
- **Headline**: specific, publish-ready title
- **Framework**: which framework generated it
- **Angle summary**: 1-2 sentences on the core argument
- **Target audience**: who this is for
- **Value proposition**: what the reader gets

### Step 4: Recommend the Top Angle

After all angles, recommend one as the strongest. One sentence explaining what makes it differentiated from what's already being published this week.

## Anti-Patterns

- Don't generate variations on the same theme - each angle must come from a different framework
- Don't output generic hooks ("Here's what you need to know about X") - every headline should be specific enough to be wrong
- Don't skip the framework label - it's what forces variety
